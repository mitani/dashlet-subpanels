Roles

Use this page to search and view existing roles in your organization. A role defines a set of access privileges to perform specific tasks. You can assign one or more users to a role to enable them to access the appropriate Sugar modules.

The Roles Home page displays the following information:

  • A Search sub-panel where you can enter the role name to search for a specific team.
  • A Shortcuts menu that displays the following options:
    • List Roles. Click this option to display a drop-down list of users. Select a user from the list to view the role and associated access permissions granted to the user in Sugar.
    • List Roles by User. Click this option to view access permissions for a specific user, and select the user from the adjacent drop-down list.
    • Create Role. Click this option to create a new role. Enter the role name and description and click Save to create the role.
  • A list of existing roles along with their description.

    • Sugar provides the following pre-defined roles for your use. You can modify these roles or create new roles to suit your requirements.

    • Sales Administrator: This role grants administration and developer privileges for Accounts, Contacts, Forecasts, Forecast schedule, Leads, Opportunities, and Quotes.
    • Marketing Administrator: This role grants administration and developer privileges for Accounts, Contacts, Leads, Campaigns, Targets, and Target Lists.
    • Customer Support Administrator: This role grants administration and developer privileges for Accounts, Bug Tracker, Cases, Contacts, and Knowledge Base.
    • Tracker: This role grants access permissions to create and manage tracker reports.

    To view details of role permissions, click the role name in the Roles list.
