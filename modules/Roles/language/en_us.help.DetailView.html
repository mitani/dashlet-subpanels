Roles

Use this page to view and set access permissions for the selected module.

Set module-level permissions as follows:

  • In the Access column, click Not Set and select Enabled to allow access, or Disabled to hide the module.
  • In the Access Type column, select of the following:
    • Not Set: to ensure that the role does not affect a particular setting.
    • Normal: to grant permissions to view and manage records depending on team membership.
    • Admin: to grant permissions to view and administer all records in the module regardless of team membership.
    • Developer: to grant developer privileges for the specified module. This allows the user access to the administration and developer tools, namely Studio, Workflow Management, and Dropdown Editor. However, the user needs the appropriate team membership to view and manage records.
    • Admin & Developer: to grant developer privileges for the specified module. This allows the user access to the administration and developer tools, namely Studio, Workflow Management, and Dropdown Editor. The user does not need team membership to view and manage records.
  • In the Delete column, click Not Set and select one of the following:
    • Select All to allow any user to delete records.
    • Select Owner to allow only record owners to delete records.
    • Select None to prevent all users from deleting records.
    Similarly, set permissions to edit records, export records, and import records. Set permissions for ListView in the List column. Set permissions for the DetailView in the View column.

    Click Save to save your changes.
