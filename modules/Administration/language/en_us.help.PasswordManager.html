Password Management

Use this page to create and manage system-generated passwords and password rules for users.

Password Requirements: Specify requirements for user-generated passwords, such as number of characters, uppercase and lowercase characters. To enforce password rules, you can use regular expressions. In the Regex Requirement field, enter specific characters and words that are not permitted in passwords.

Email Templates: You can create and manage templates to send system-generated passwords and links to reset passwords to users. To create an email template to send out system-generated passwords, select System-generated password email from the drop-down list.To create an email template to send a link to users who forgot their passwords, select Forgot Password email from the drop-down list.

Generated Link: Specify an option to manage the system-generated link that allows users to reset their passwords. Select None if you do not want the link to expire; select Link Expires in, and enter the time period, in minutes, hours, or days, when the link remains active.

User-Generated Password Expiration: Select one of the following:

  • None. The password never expires.
  • Password Expires in. The time period, in days, weeks, and months, after which the password expires.
  • Password Expires after how many login attempts? The number of times a user can log into Sugar. When this number is met, the password expires and the user must specify a new password.

The last two options listed above ensure that users change their passwords periodically for the security of your Sugar application.

Login Lockout: You can specify whether user-generated passwords expire or not. If you choose to force users to change passwords, you can choose a time period or the number of logins.

Captcha Validation: You can enable captcha validation for additional security. Checkmark the box and enter the Public key and the Private Key that you received from reCAPTCHA for your Sugar instance.

Click Save to save settings.
