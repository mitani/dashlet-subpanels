System Locale Settings

Use this page to configure system-wide defaults for date, time, language, currency, and salutation formats for your organization.If you are using MySQL, you can also specify the collation order for records in the application.

In the User Interface sub-panel, specify the following information:

  • Default Date Format. From the drop-down list, select a date format for all records such as Quotes and Contracts.
  • Default Time Format. From the drop-down list, select a time format to display in all records such as Cases.
  • Default Language. From the drop-down list, select the default language for the Sugar User Interface.
  • Default Name Format. Enter the default salutation and name format to display in list views and detail views.

In the Default Currency sub-panel, specify the following information:
  • Currency. Enter the name of the currency that your organization uses to conduct business.
  • Currency Symbol. Enter the symbol for the currency.
  • ISO 4217 Currency Code. Enter the ISO code for the currency.
  • 1000s Separator. Specify a delimiter to separate thousands when users specify a numeric value for the amount.
  • Decimal Symbol. Specify a default decimal symbol.

In the Export Settings sub-panel, specify the following information:
  • Export Delimiter. Specify the delimiter to separate data fields during export.
  • Default Export Character Set. From the drop-down list, select the character set to use during export.
  • Disable Export. Select this option to disallow data export from Sugar.
  • Admin Export Only. Select this option to allow only users with administrative privileges to export data.

In the Database Collation sub-panel, specify the following information:
  • Collation. To change the default database collation from utf8_general_ci, select the new value from the drop-down list.
Click Save to save the settings.
